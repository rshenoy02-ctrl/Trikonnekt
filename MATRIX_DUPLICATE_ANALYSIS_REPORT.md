## Matrix Creation Rules (Confirmed)

Each transaction source should create **1 FIVE_150 + 1 THREE_150 account** per occurrence:

### 1. **Promo Purchases**
- **PRIME150** (₹150): Each purchase creates 1 FIVE + 1 THREE
- **PRIME750+** (₹750+): Only FIRST approved purchase for each package code creates matrices (subsequent purchases for same package do NOT create new matrices)
- **MONTHLY759**: Does NOT create matrices (distribution-only)

### 2. **Self Account Allocations** (NEW - 208 instances found)
- Triggered by SELF_ACCOUNT_DEBIT transaction with source_type='SELF_250_PACK'
- Amount: ₹250 per allocation
- Effect: Each allocation internally calls distribute_150_prime which creates 1 FIVE_150 + 1 THREE_150
- **These are legitimate matrix-creation sources that were NOT in original analysis**

### 3. **E-Coupon 150 Activations** (Not found in current window)
- CouponSubmission with status='AGENCY_APPROVED' and value=150
- Each creates 1 FIVE_150 + 1 THREE_150
- Currently 0 found in 60-day window
