# CORRECT MODEL: 5-MATRIX (Sponsored) + 3-MATRIX (Global)

## User1 Buys PROMO Package = Creates 2 Separate Matrix IDs

When user buys promo (250 self allocation):
- Creates **1 NEW 5-MATRIX ID** (FIVE_150 pool) - Sponsored level tree chain
- Creates **1 NEW 3-MATRIX ID** (THREE_150 pool) - Global level (fills empty slots)
- Both are **COMPLETELY SEPARATE** earning streams

## IMPLEMENTATION NOTES

### For Backend (models.py):
```
1. When PROMO package purchased:
   - Call create_five_150_for_user(User1, pool_type='FIVE_150')
     └─ Creates ID with parent = sponsor's positioned account
   - Call create_three_150_for_user(User1, pool_type='THREE_150')
     └─ Finds next empty global slot via MLM algorithm
     └─ Creates ID with parent = that global slot

2. Each pool type is completely independent:
   - 5-MATRIX algorithm: BFS under sponsor chain
   - 3-MATRIX algorithm: BFS under global slot
   - No mixing of tree structures

3. Commission calculation:
   - Query: WHERE owner=User1 AND pool_type='FIVE_150' → get earnings
   - Query: WHERE owner=User1 AND pool_type='THREE_150' → get earnings
   - Sum both = total User1 earnings
```

### For Frontend (Genealogy):
```
1. When viewing User1's tree:
   - Show both matrix tabs
   - 5-MATRIX tab: visualize 5-position tree (User2-6, spillover)
   - 3-MATRIX tab: visualize 3-position tree (User7-9, spillover)
   
2. Earning dashboard:
   - 5-MATRIX earnings: From pool_type='FIVE_150' tree
   - 3-MATRIX earnings: From pool_type='THREE_150' tree
   - Total: Sum of both
```
